Установка

Установка OpenCart

В этом руководстве будет описано, как успешно установить и удалить OpenCart.

Видеоурок

Скачивание и распаковка архива OpenCart

Последнюю версию OpenCart (v.3.0.2.0) можно скачать с официального сайта OpenCart (рекомендуется для конечного пользователя) или напрямую с Github (рекомендуется для веб-разработчиков). Страница загрузки также предоставляет доступ к предыдущим версиям OpenCart. В колонке Downloads найдите 3.0.2.0 и нажмите на ссылку "Download Now", расположенную напротив неё в колонке Action. Это загрузит сжатый архив данной версии OpenCart в виде zip-файла. Для версии OpenCart 3.0.2.0 будет загружен файл с именем "3.0.2.0-compiled.zip".

Если у вас нет программы на компьютере, способной извлекать файлы из zip-файла, вы можете скачать WinRar бесплатно. Распаковка zip-файла позволит получить доступ к архиву OpenCart, чтобы файлы могли быть доступны веб-серверу.

При открытии zip-файла будут видны следующие файлы:

  • "upload" папка
  • "license.txt" файл
  • "readme.txt" файл

Папка "upload" содержит все файлы, необходимые для загрузки OpenCart на веб-сервер. Файл "license.txt" содержит лицензионное соглашение относительно использования OpenCart на вашем сайте. Файл "readme.txt" предоставляет ссылки на текущие инструкции по установке и обновлению на сайте OpenCart. Когда вы будете готовы, вы можете извлечь файлы из zip-файла в любое место на вашем компьютере.

Загрузка OpenCart

На этом этапе у вас должен быть установлен веб-сервер и извлечён архив OpenCart. Теперь мы будем использовать клиент File Transfer Protocol (FTP) для загрузки этих распакованных файлов на веб-сервер.

Установка и настройка FTP-клиента

Рекомендуем использовать Filezilla в качестве вашего FTP-клиента. Filezilla — это бесплатный FTP-клиент, который позволяет передавать файлы OpenCart на любой указанный веб-сервер. Клиент FileZilla (не сервер) можно скачать с https://filezilla-project.org/ и установить на компьютер.

Когда вы откроете Filezilla, вы должны увидеть файловую директорию вашего компьютера на левой стороне. Следующий шаг — найти, куда вы сохранили распакованный архив OpenCart, и нажать на папку "upload", файлы должны появиться ниже. Директория должна оставаться открытой, пока мы продолжаем. Правая сторона в данный момент пустая, потому что целевой веб-сайт не подключен. При подключении отобразится файловая директория веб-сервера.

Перед тем как продолжить, необходимо убедиться, что у вас есть следующая информация о вашем сайте:

  • имя хоста
  • имя пользователя
  • пароль

Эту информацию можно получить, обратившись к вашему хостинг-провайдеру.

Подключение к Менеджеру сайтов

В меню “File” выберите “Site Manager” в Filezilla. Откроется окно с открытой вкладкой General. Вам нужно заполнить информацию, собранную выше, о хостинге, и нажать “Connect”. Теперь правая сторона “Remote Site” отобразит файловую директорию вашего веб-сайта.

Загрузка файлов OpenCart

Если вы еще не нашли папку загрузки OpenCart на левой стороне, сделайте это сейчас и оставьте ее открытой. В директории Remote Site (правой стороне) откройте папку, в которой будет находиться магазин OpenCart. Местоположение магазина зависит от того, хотите ли вы, чтобы магазин отображался на главной странице, в подпапке, поддомене и т.д. Если вы хотите, чтобы OpenCart был главной страницей, вам нужно загрузить файлы в корневую папку вашего веб-сайта.

Имейте в виду, что некоторые хостинг-сервисы требуют загрузки общедоступных файлов в общедоступную директорию, такую как public_html, чтобы они были видны на сайте. Вам следует уточнить у вашего хостинг-провайдера, куда вы можете загружать общедоступные файлы.

После определения местоположения магазина OpenCart необходимо выбрать весь контент из папки “upload” на левой стороне Filezilla, щелкнуть правой кнопкой мыши и загрузить. Загрузка всех файлов может занять несколько минут на FTP-клиенте.

Если вы хотите, чтобы магазин находился на главной странице, например www.shopnow.com, вы должны загрузить содержимое папки “upload”, но не саму папку “upload”. Если включить папку “upload”, это создаст подпапку, и магазин будет доступен только по адресу www.shopnow.com/upload.

После завершения загрузки файлов Filezilla в указанное место вы должны увидеть одинаковые файлы с обеих сторон: на левой стороне (компьютере) и на правой стороне (веб-сайте), как показано на скриншоте ниже:

cPanel

Если ваш веб-хостинг использует cPanel, вы можете попробовать этот метод без FileZilla.

Перейдите в папку “Upload” и выберите все файлы внутри, затем сожмите их в новый zip-файл. Войдите в ваш cPanel и нажмите на “File Manager”, чтобы открыть новую вкладку в браузере. Загрузите новый zip-файл в целевой путь, он должен находиться внутри папки public_html. После этого вы можете щелкнуть правой кнопкой мыши на zip-файл и выбрать кнопку “Extract”, и всё готово.

Создание базы данных для магазина

Следующий шаг — создать базу данных на сервере MySQL, чтобы OpenCart мог хранить данные магазина. Вам нужно войти в панель управления сайта и найти раздел MySQL Databases. Используя MySQL Databases, вы можете создать новую базу данных, введя имя базы данных и имя пользователя/пароль для доступа к этой базе данных. Созданный пользователь должен быть добавлен в базу данных с включением всех необходимых прав. Мы будем использовать эту информацию о базе данных позже, когда будем настраивать OpenCart с помощью автоинсталлятора.

Запуск автоинсталлятора

С новой базой данных, созданной только что, мы готовы установить OpenCart непосредственно на веб-сайт. Откройте веб-браузер и введите веб-адрес, куда вы загрузили OpenCart. Если папка «install» в папке «upload» была загружена корректно, вы должны автоматически увидеть следующую страницу:

Эта страница является страницей установки. Следующие шаги помогут вам завершить процесс установки OpenCart.

Шаг 1. Лицензия

Вы можете ознакомиться с лицензией, отметить "Я согласен с лицензией" и нажать “Продолжить”.

Шаг 2. Предварительная установка

Этот шаг проверяет, соответствует ли веб-сервер техническим требованиям и имеет ли правильные права для корректной установки OpenCart. Если видны красные отметки на пунктах #1 или #2, вам нужно убедиться, что эти компоненты правильно установлены на веб-сервер. В случае возникновения проблем обратитесь в службу поддержки вашего хостинга. Если красные отметки на пунктах #3 или #4, возможно, возникла проблема с загрузкой файлов с помощью FTP-клиента. Если всё настроено правильно и видны зелёные отметки (как показано ниже), вы можете нажать "Продолжить".

Начиная с версии 2.3.0.2, следующие файлы необходимо переименовать, чтобы продолжить к Шагу 3:
  • config-dist.php config.php
  • admin/config-dist.php config.php

Шаг 3. Конфигурация

Вам нужно заполнить поля под пунктом #1, добавив данные для подключения к базе данных (имя пользователя/пароль, имя базы данных), которые вы установили при создании базы данных магазина с помощью MySQL Databases. В пункте #2 вы должны создать новое имя пользователя и пароль для доступа к административной панели OpenCart. Когда закончите, нажмите "Продолжить".

Шаг 4. Завершение установки

Теперь, когда установка завершена, папка "install" больше не требуется. Вы должны удалить папку “install” в разделе Remote Site в Filezilla по соображениям безопасности. Теперь у вас есть возможность просмотреть фронтенд вашего магазина или войти в административную панель.

После установки вам следует предпринять необходимые шаги для повышения безопасности вашего сервера и директории магазина OpenCart. Эти шаги подробно описаны в Основные практики безопасности.

Установка vQmod

vQmod для OpenCart позволяет вам добавлять модули в ваш магазин без изменения исходного кода OpenCart. Для получения дополнительной информации о том, как скачать и установить vQmod для вашего магазина, смотрите.

Удаление OpenCart

Удаление OpenCart — это быстрый и простой процесс.

  1. Удалите файлы/папки в месте, куда вы их загрузили в Filezilla.
  2. Удалите базу данных OpenCart из MySQL Databases или phpMyAdmin.
После удаления OpenCart вся информация о товарах и клиентах будет утеряна. Вы можете сохранить резервную копию базы данных на случай, если захотите переустановить OpenCart позже. Для получения дополнительной информации смотрите Резервное копирование/Восстановление.

Поддержка
