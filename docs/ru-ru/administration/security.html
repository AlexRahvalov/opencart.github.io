Основные практики безопасности

OpenCart не несет ответственности за безопасность вашего веб-сайта, поэтому именно вы должны обеспечить уровень безопасности вашего сервера. Следующие рекомендации направлены на улучшение безопасности вашего магазина OpenCart.

Эти дополнительные меры могут быть предприняты сразу после установки OpenCart на ваш сервер или когда ваш магазин становится активным. См. Установку для получения дополнительной информации о том, как установить ваш магазин OpenCart.

Удалите папку установки

OpenCart настоятельно рекомендует удалить папку установки сразу после завершения установки. OpenCart предупредит вас в администраторе, если папка установки не была удалена.

Защита каталогов

Папка администратора

Папка администратора — это место, где вы получаете доступ к администрированию вашего магазина. Люди с доступом к администрированию вашего магазина смогут редактировать ваши продукты, информацию о клиентах, настройки магазина и другую ценную информацию. Поэтому очень важно, чтобы вход в админку был труднодоступен для посторонних.

Переименование администратора

Переименование папки администратора на что-то, не относящееся к администратору, необходимо для предотвращения нежелательного доступа к её расположению. Вы можете получить доступ к вашему входу в админку, введя местоположение вашего магазина, а затем путь к администратору. Например, если папка администратора была изменена на "cookiemonster", вход в админку будет по адресу "www.yourstore.com/cookiemonster". Вам также нужно будет обновить файл admin/config.php (или в примере — файл cookiemonster/config.php), заменив все вхождения "admin" на новое имя, например, "cookiemonster". Обратите внимание, что это может усложнить вам жизнь в будущем, например, если вы решите установить vQmod.

.htaccess и .htpasswd

Файлы .htaccess и .htpasswd в папке администратора предотвратят доступ хакеров к вашему магазину, даже если они обнаружат местоположение входа в админку. С помощью .htaccess вы можете запретить всем IP-адресам просмотр вашего магазина, кроме IP-адреса администратора. Файл .htpasswd в папке администратора потребует дополнительный пароль для доступа разрешённому администратору к этой директории.

Каталог

Каталог можно защитить с помощью традиционного файла .htaccess. Использование FileMatch может быть полезно для защиты важных типов файлов вашего магазина, таких как .php и .txt, а не всех файлов. Следующий код можно использовать для .htaccess в вашей папке каталога:

<FilesMatch "\.(php|twig|txt)$">
Order Deny,Allow
Deny from all
Allow from "ваш ip адрес"
</FilesMatch>

Это запретит доступ ко всем файлам .twig, .php и .txt.

Папка системы

Папка системы содержит два файла, которые нужно защитить: logs/error.txt и start_up.php. Файл logs/error.txt при необходимости можно переименовать.

.Htaccess

Файл .htaccess будет работать для защиты этих файлов и подпапок системы от доступа посторонних, кроме назначенного администратора. Для этого вставьте следующий код в ваш .htaccess:

<Files *.*>
Order Deny,Allow
Deny from all
Allow from "ваш ip адрес"
</Files>

Права на файлы

Следующие файлы необходимо установить на 644 или 444, чтобы предотвратить запись в них посторонних:

  • config.php
  • index.php
  • admin/config.php
  • admin/index.php
  • system/startup.php
